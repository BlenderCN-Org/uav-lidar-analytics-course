Data

  • North Carolina data set used in MEA582
  • boundaries of COA in NC obtained by NGAT
  • sUAS surveys of LW area:
    images, log file from the flight 06/20/2015, DSMs
  • raw text format of GCP locations
  • GRASS location containing timeseries of DSMs computed from example flights, includes GCPs and COA layers
  • 2013 lidar data (Wake county survey):
    • secref_spm_2013.las and mid_pines_spm_2013.las, and already interpolated DEM/DSM (use r.unpack)
    • LAS 2013 tiles overview is available as KML file LAS_overview_2013.kml or GRASS vector file LAS_overview_2013.pack for nc_spm.
    • Additional 2013 LAS files will be posted here.
      Use las2las --t_srs=EPSG:32119 -i input_spf.las -o output_spm.las to reproject it.
  • 2001 lidar data (NC Floodplain mapping - for SECREF it is in the MEA582 data set)
  • data not provided elsewhere are available through Google Drive (accessible with NCSU account)

Software

  • GRASS GIS 7.0.1 (you can use these instructions to install it)

  • open source software MicMac (documentation can be found here)
  • software provided at NCSU is available through Google Drive (accessible with NCSU account)

Assignments

  1. Explore different types of UAS
  2. Flight plan preparation - use Trimble Aerial Imaging (Assignment 2a) and GIS (Assignment 2b)
  3. Processing the data to create orthophoto and DSM
  4. Comparing DSMs processed using different workflows (assess the importance of GCPs, GPS and IMS accuracy, SfM only)
  5. Multiple return lidar data analysis and visualization
  6. Fusion of lidar and UAV data
  7. Management and visualization of multitemporal data in GRASS GIS 7
  8. Geospatial simulations of dynamic processes at ultra high resolution: impact of terrain change
    • Water flow
    • Solar radiation
  9. Processing and analysis of elevation series with GRASS GIS and Python
    • Introduction to Python (additional resource: Codecademy Python track)
    • Python scripting in GRASS GIS
    • Working with individual points
    • Creating a script
    • Homework: Create a script to compare a set of DEMs (time series or differently acquired data)
